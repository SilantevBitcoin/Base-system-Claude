# Этап 0 — От мысли до идеи

> Ты здесь, когда у пользователя **сырая мысль** («хочу что-то с X», «а что если…») и неясно, стоит
> ли вообще это делать.

**Открой [`../spravochniki/idea-kb.md`](../spravochniki/idea-kb.md)** и веди по нему: расширить
(diverge) → отсеять (converge) → обязательный **гейт go / no-go / reframe**. Критерии отсева,
принципы и anti-sycophancy — в справочнике.

**Граница (важно):** только ЧТО / ДЛЯ КОГО / РЕАЛЬНА ЛИ БОЛЬ / СТОИТ ЛИ. Архитектуру и «как» НЕ
трогаешь — это этап 1 (brainstorming).

**Конец этапа — валидация рынком (для продуктовых идей, перед гейтом):** боль и ландшафт — скилл
`<MARKETING>/skills/customer-research/SKILL.md` (Reddit/Quora/G2: боль реальна? кто уже решает?);
реалистичность каналов — `<MARKETING>/skills/marketing-ideas/SKILL.md` (139 тактик). Результат кормит
гейт. Набор marketingskills — см. `~/.claude/rules/marketing.md`.

**Выход:** `idea-doc` + вердикт. На `go` → [`../1-plan/`](../1-plan/README.md). На `no-go` →
зафиксируй причину (honest kill — валидный результат).

Назад в оглавление: [`../CLAUDE.md`](../CLAUDE.md)
