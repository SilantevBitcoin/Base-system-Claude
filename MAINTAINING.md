# Поддержка публичной копии — как готовить к публикации

> **Кому.** Мейнтейнеру (владельцу системы или его ИИ-ассистенту), который обновляет ЭТУ публичную
> копию из рабочей (приватной) системы. Пользователю репозитория это не нужно — ему `README.md`.

Эта папка — **публичная копия** рабочей dev-системы, подготовленная к публикации: без личных путей,
секретов и конкретики окружения. Рабочая система живёт отдельно (приватно); её изменения переносятся
сюда **вручную с адаптацией**, не автоматическим зеркалированием (так чувствительное не утечёт случайно).

## Когда обновлять

После того как в рабочей системе появились новые модули/направления или изменились справочники,
правила, маршруты, хук. Признак — рабочая система ушла вперёд этой копии.

## Что перенести (найти расхождения)

1. В рабочем репозитории: `git log` после последнего `sync`-коммита этой копии → список изменённых
   файлов по модулям (`git show <commit> --stat`).
2. Сравнить по областям: `spravochniki/`, этапы (`0-ideya`…`3-soprovozhdenie`), `meta/`, «руки»
   (`agents/`, `rules/`, хук-скрипты, `settings`).

## Правила адаптации (обязательно перед публикацией)

| В рабочей системе | В публичной копии |
|---|---|
| Личный путь (`D:\…`, `C:/Users/<имя>`) | относительный путь, либо плейсхолдер `<PLACEHOLDER>` + install-инструкция |
| Внешний клон по личному пути (напр. набор скиллов) | плейсхолдер (`<MARKETING>` и т.п.) + как склонировать (`git clone …`) |
| `~/.claude/...` | **оставить** — это стандартный путь «рук», одинаков у всех |
| Имя владельца в функциональных правилах | «владелец» (ФИО — только в `LICENSE` / `README` / `NOTICE` как автор) |
| Личные хуки (звук, авто-pull, статуслайн), `model`, `defaultMode`, `additionalDirectories` | НЕ переносить в `settings.example.json` |
| Плагины (ставятся из маркетплейсов) | НЕ в репо — только install-инструкция (`/plugin install …`) |

«Руки» в этой копии лежат в `runtime-copy/` (`agents/`, `rules/`, хук-скрипты, `settings.example.json`)
как набор для установки в `~/.claude/`. Это просто папка-контейнер для раздачи — не зеркало, не источник правды.

## Сверка перед публикацией

```bash
# личных путей/имени/секретов быть НЕ должно (кроме плейсхолдеров <…> и автора в LICENSE/README/NOTICE):
grep -rnE 'C:/Users/[a-z]|<реальное-имя-владельца>' --include='*.md' --include='*.json' --include='*.py' . | grep -v '/.git/'
# JSON валиден:
python -m json.tool runtime-copy/scripts/dev_routing.json
python -m json.tool runtime-copy/settings.example.json
```

## Публикация

Личная почта НЕ должна попадать в публичную историю — автор коммитов под **noreply**-адресом GitHub:

```bash
git config --local user.email "<id>+<user>@users.noreply.github.com"
git config --local user.name  "<Имя автора как в LICENSE>"
git add -A
git commit -m "sync: <что перенесено>"
git push origin main
```
