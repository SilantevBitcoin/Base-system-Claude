# Диспетчер ленивой загрузки — как устроен и как менять

> **Кому.** Агенту, который будет расширять dev-систему (добавлять направление/колонку,
> чинить или настраивать загрузку правил). Прочти перед правкой хука или `rules/`.
>
> ⛔ Это файл системы. Меняй только по прямой просьбе владельца (`~/.claude/rules/protect-dev-system.md`).

## Зачем это

Раньше **все** `~/.claude/rules/*.md` грузились в КАЖДУЮ сессию — включая привязки всех направлений,
даже когда задача не про код. Постоянный налог на контекст.

Теперь — **трёхслойная ленивая загрузка по словам в чате**. Грузится только то, что нужно
под текущую задачу. Экономия: не-код сессия ~6k вместо ~17k; код-сессия ~9k.

## Три слоя (модель «мозг / хребет / руки»)

| Слой | Что | Когда грузится | Механизм |
|---|---|---|---|
| **Мозг** | оглавление Base (`CLAUDE.md`) + поведенческие `rules/*.md` | всегда | авто-загрузка Claude Code |
| **Хребет** | `coding-discipline.md` + процесс superpowers | стадия план/код (не идея) | хук вкладывает |
| **Руки** | `rules/<направление>.md` + роль-агент + скиллы | направление на реализации | хук вкладывает |

Триггер всех переключений — **слова пользователя в чате**, не открытие файлов.

## Карта файлов

```
~/.claude/
├── settings.json                  → "claudeMdExcludes": снимает coding-discipline + направленческие
│                                     rules с авто-загрузки. Хук в UserPromptSubmit.
├── rules/                         → ФИЗИЧЕСКИ все на месте (ссылки в маршрутах не ломаются!)
│   ├── <поведенческие>.md           грузятся всегда (НЕ в claudeMdExcludes)
│   ├── coding-discipline.md         в excludes → вкладывается хуком по стадии
│   └── frontend/typescript/python/db/ai/backend/marketing/design.md   в excludes → по направлению
└── scripts/
    ├── dev_dispatcher.py          → хук-движок (читает stdin-промпт, решает что вложить)
    └── dev_routing.json           → ТАБЛИЦА-ДАННЫЕ: стадии + направления (правишь ТОЛЬКО её)
```

Ключевой приём: файлы НЕ перемещены из `rules/`. `claudeMdExcludes` убирает их из
авто-загрузки, оставляя на диске → хук читает их оттуда, а все ссылки `rules/<dir>.md`
в `orchestrator.md` / README этапов / агентах остаются рабочими.

## Как работает (поток)

1. Пользователь пишет сообщение → `UserPromptSubmit` запускает `dev_dispatcher.py`.
2. Хук читает `dev_routing.json`, по словам промпта определяет **стадию** (build > plan > idea)
   и **направления** (по keywords + расширениям файлов).
3. Собирает `additionalContext`:
   - стадия план/код ИЛИ есть направление → вкладывает `coding-discipline.md` (хребет);
   - стадия build, либо план/без-стадии с названным направлением → вкладывает `rules/<binding>`
     + строку «роль-агент + скиллы» (руки);
   - стадия idea → только процесс brainstorming, без хребта и рук;
   - всегда → напоминание karpathy-guidelines.
4. Этот текст уходит ТОЛЬКО в контекст модели (не в видимый транскрипт).

**Bias:** при неоднозначности грузим чуть больше (лучше лишнее, чем пропуск).

## Как добавить новую колонку/направление (например, SEO)

Три шага, хук менять НЕ нужно:

1. **Создай правило-привязку** `~/.claude/rules/seo.md` — таблица «стадия → инструмент» для SEO
   (по образцу `python.md`). Это «руки» направления.
2. **Добавь запись в `dev_routing.json` → `directions`:**
   ```json
   "seo": {
     "keywords": ["seo", "копирайтинг", "мета-тег", "ключевые слова", "сниппет", "title-тег"],
     "ext": null,
     "binding": "seo.md",
     "agent": "seo-writer",
     "skills": ["keyword-research", "serp-analysis"]
   }
   ```
3. **Добавь `seo.md` в `claudeMdExcludes`** (`settings.json`), чтобы не грузился eager:
   `"**/.claude/rules/seo.md"`.

Готово: фразы вроде «напиши SEO-описание» начнут подтягивать `seo.md` + роль-агента автоматически.
(Опц.) пропиши направление в оглавление Base и роль-агента в `~/.claude/agents/` — если колонка
полноценная, как остальные.

## Как менять стадии

Стадии (`idea` / `plan` / `build`) и их ключевые слова — в `dev_routing.json → stages`.
Приоритет фиксирован в коде хука: `build > plan > idea` (есть слово «напиши» → реализация,
даже если рядом «план»). Добавить слово-триггер = дописать в `keywords` нужной стадии.

## Как проверить после правки

```bash
cd ~/.claude/scripts
export PYTHONUTF8=1
echo '{"prompt":"<твой тест-промпт>"}' | python dev_dispatcher.py \
  | python -c "import json,sys; r=json.load(sys.stdin)['hookSpecificOutput']['additionalContext']; print('\n'.join(l for l in r.splitlines() if l.startswith(('[dev]','[hook]','=====')))); print('len:',len(r))"
```
Маркеры `===== файл =====` показывают, что реально вложилось. `dev_routing.json` должен быть
валидным JSON (`python -m json.tool dev_routing.json`).

Эффект `claudeMdExcludes` (что direction-rules больше не грузятся eager) виден ТОЛЬКО в новой
сессии — проверяется командой `/memory` (исключённых файлов не должно быть в списке загруженных).

## Откат

- Вернуть авто-загрузку: убрать файл(ы) из `claudeMdExcludes` в `settings.json`.
- Вернуть прежний хук: восстановить предыдущую команду `UserPromptSubmit` в `settings.json` из бэкапа.
