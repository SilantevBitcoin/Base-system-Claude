# Этапы 5–6 — Техническая реализация (код + доки по ходу)

> Ты здесь, когда на руках PRD/дизайн/план и пора писать код. Один цикл и для нового кода, и для
> правок готового. Доки по ходу (этап 6: ADR на решения, `knowledge.md` на грабли) — часть этого же
> цикла, не отдельный заход (линза D спины).

**Спина — [`../spravochniki/dev-coding-process-kb.md`](../spravochniki/dev-coding-process-kb.md):**
стадии 0–6 (Сориентироваться → Оформить → Написать(TDD) ⇄ Проверить ⇄ Отладить → Ревью → Завершить),
ядро — петля 2⇄3⇄4. Дисциплина, стандарт «готово», безопасность — в справочнике и в рельсе
`~/.claude/rules/coding-discipline.md`.

## Направления — руки в `~/.claude/`

**Объяви направление** — привязки всех направлений уже автозагружены в сессию; объявление выбирает,
по какой таблице (`~/.claude/rules/<направление>.md`) действовать → роль-агент ведёт стадии и дёргает
скиллы как инструменты. Гейты «готово» каждого направления — в его `rules/<dir>.md`:

| Направление | Роль-агент | rules |
|---|---|---|
| фронт (React/UI) | `frontend-engineer` | `rules/frontend.md` |
| TS/Node (основной app) | `typescript-engineer` | `rules/typescript.md` |
| Python (воркеры/AI) | `python-engineer` | `rules/python.md` |
| БД | `dba` | `rules/db.md` |
| AI/данные | `data-scientist` · `mlops` | `rules/ai.md` |
| бэк (distributed/инфра) | app-код: `typescript-engineer`/`python-engineer`; инфра-дисциплина: `devops-engineer` (НЕ пишет бизнес-логику) | `rules/backend.md` |

**Кто пишет код — решает владелец.** Перед стартом кодового куска главная сессия коротко предлагает
режим с рекомендацией: (а) **малое/простое** → пишет сама, неся дисциплину роли (роль-агент = таблица
и гейты, не субагент); (б) **большой объём / несколько направлений / риск забить контекст** →
роль-агенты как субагенты (Agent tool; контракты на стыках первыми — см. спину). Выбор за владельцем.

Полиглот-проект задействует несколько направлений (напр. SaaS = frontend + typescript + dba + backend).
**Кто ведущий и как свести направления** (главная сессия = tech-lead · контракты на стыках первыми · сквозное «готово») —
в спине, раздел [«Координация полиглот-фичи»](../spravochniki/dev-coding-process-kb.md#координация-полиглот-фичи-кто-ведущий).

**Выход:** рабочий код + тесты, гейты зелёные. → Ревью и финал в
[`../3-soprovozhdenie/`](../3-soprovozhdenie/README.md).

Назад в оглавление: [`../CLAUDE.md`](../CLAUDE.md)
