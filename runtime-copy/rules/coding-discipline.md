# Дисциплина кода — общий рельс (A + B)

> Стек-агностичная дисциплина под ВСЕ направления. `rules/<направление>.md` (frontend/typescript/
> python/db/ai/backend) добавляют специфику и гейты поверх этого рельса. Спина стадий 0–6 — в
> справочнике системы `spravochniki/dev-coding-process-kb.md`. Объявил направление → его `rules/` + роль-агент
> ведут стадии, дёргая скиллы. Этот файл = что обязательно на любом коде, в любом направлении.

## A. Дисциплина написания (Moves — процедура, не личность)

**Mindset (karpathy):** думать до кода · простота (минимум, что решает задачу) · surgical (трогать только
нужное) · цель→проверка (задача = проверяемый критерий, не «сделать работающим»).

**Move 1 — Слой до первой строки.** Назови слой (core/domain/infra/handlers), куда ложится код.
Зависимости — внутрь; наружу не импортируем. Не можешь назвать слой одним словом → стоп, определи.

**Move 2 — Контракт раньше тела.** Сигнатура + типы, pre/post-условия одной фразой, инвариант для
сайд-эффектов, объявленный случай ошибки в типе возврата (Result/typed error, не «голый» throw для
ожидаемого). Тело раньше сигнатуры — запрещено. Каждый шаг тела локально следует из контракта.

**Move 3 — 7 запретов (обход — только с обоснованием в месте использования):** глобальный mutable
state · monkey-patch · reflection для потока управления · исключения для ожидаемого · aliasing ·
dynamic dispatch с неизвестным телом · «умные» однострочники. + нетипизированный `any`/`dict` через
границу слоя.

**Move 4 — Причина в корне.** Репро (нет репро — нет фикса) → инструментировать до гипотезы → измерить
по всем правдоподобным компонентам → бисект до носителя → подтвердить вторым методом. Причина ровно одна
из: (a) неверный контракт · (b) нарушение слоя · (c) спутанные concerns · (d) конструкт убил локальное
рассуждение · (e) дисциплина ниже ставок. Чинить в источнике, не заплатку на месте симптома.

**Move 5 — Разделение concerns.** Корректность сложной функции = произведение корректностей частей.
Две заботы в одной функции (I/O vs вычисление, policy vs mechanism) → разделить, общаться через
интерфейсы/чистые данные, не через shared mutable state.

**Move 6 — Самопроверка до отгрузки.** Дисциплина соблюдена · контракты на месте · слой не нарушен ·
запреты обоснованы или отсутствуют · тесты зелёные (по ставкам) · назвать топ-3, что ещё может
опровергнуть изменение. Провал прохода → итерировать или передать специалисту.

**Move 7 — Строгость по ставкам (классификация объективная, не самоназначенная):**
- **High** — auth/деньги/данные/crypto/concurrency/миграции · файл >500 строк · импортируется >5
  модулями · >1 автор за 90 дней. → полная дисциплина (Moves 1–5) + тесты обязательны.
- **Medium** — бизнес-логика или прод вне High. → Moves 1,2 на границах, 3, 4; Move 5 на стыках.
- **Low** — скрипты/прототипы/UI-косметика. → Moves 1, 3; остальное неформально. Метка «прототип»
  истекает через 30 дней или на первом прод-импорте.

  Moves 1 и 3 — на всех уровнях. Классификацию назвать в выводе; не обоснована → дефолт Medium.

  **Blast-radius до правки существующего символа** (смена сигнатуры / удаление / переименование) — LSP
  `incomingCalls`/`findReferences` на символе = карта «кого сломаю»; она же измеряет критерий High
  «импортируется >5 модулями». Первый вызов в сессии = холодный индекс: пустой/подозрительно малый ответ — повторить.

**Move 8 — Соседи перед правкой (ripple первого порядка).** Перед изменением элемента, делящего с
другими **общий ограниченный ресурс**, — секундный взгляд: кто рядом и что с ним станет. Ресурс (4 вида):
**место** (ширина/высота в строке·контейнере·сетке·flex) · **общий родитель** (соседи того же узла) ·
**общее значение/состояние** (константа·переменная·поле·проп, что читают другие) · **автоматика на
связанном** (FK-каскад·триггер·вычисляемое поле). Нежелательный эффект → выбрать путь правки, который
**гасит его сразу** (не «поменяю, потом гляну»). Порог (анти-театр): изолирован / эффект нулевой →
отработать в уме, в текст не выводить; ненулевой → одна строка «меняю X → сдвинет Y → делаю Z». Это
первый порядок, видимый глазом — не транзитивная цепочка (Move 7 на high-stakes) и не замена
проверки-по-результату. На **всех ставках**, включая Low.

## B. Стандарт «готово» (definition of done)

- **Логично** — тело следует из контракта; шаг, который трудно обосновать против pre/post, — код неверен.
- **Проверяемо** — за каждым утверждением о коде тест, замер, тип или ассерт. «Думаю, работает» —
  гипотеза, не готовность. `any` не делает утверждения.
- **По ставкам** — строгость пропорциональна последствию. Процесс-театр на Low — тоже провал.
- **Только нужное** — мёртвый код, backward-compat-shims, «на всякий», преждевременные абстракции,
  неиспользуемые экспорты — удалить. Что построено — должно вызываться.
- **Долг доказательств** — сам ищи источник и замер. Нет источника константы/факта → «не знаю», стоп.
  Уверенный неверный ответ разрушает доверие; честное «не знаю» — сохраняет.

## Отказы по умолчанию (refuse без обоснования в месте)

- Заплатка в прод (guard/null-check/try-catch на месте throw) без анализа причины (Move 4).
- Импорт из невидимого слоя (core → infra).
- «Обработка ошибок на всякий» без названного условия отказа (`// FAILS_ON: <условие>`).
- Хардкод-константа без источника (`// source: <URL|paper|benchmark|measured>`). «Works» — не источник.
- Отгрузка High без тестов.
- Правка кода, который не можешь объяснить своими словами.

## Безопасность (сквозная линза, не стадия)

Вход — поверхность угроз; письмо — запреты и валидация на границах; ревью — OWASP/CWE.
Секреты только из env, не в коде. Чувствительное (auth/деньги/данные) → `/security-review`.
