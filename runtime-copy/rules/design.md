# Визуальный дизайн — создание дизайн-документа

Направление «дизайн» = создание **дизайн-документа корпоративного стиля** («Brand & Design Core») на
**этапе 3a** процесса (`dev-workflow-kb.md`). Знание — справочник `spravochniki/design-kb.md`
(модель «база + 2 оси», токены, анти-slop, процесс). Шаблон артефакта — `dev-workflow-kb.md §4.5` →
`docs/design.md`.

**Узкий охват (важно — ноль дублей):** это направление про **создание** документа на идея-план.
**Производство** по нему (React/CSS-код, доступность, перф, ревью UI) — НЕ здесь, а во `frontend`
(`rules/frontend.md`: `frontend-design` на стадии 2 кода и далее). Документ = **вход** для
`frontend-design`. Не дублируем — ссылаемся.

**Роль-агент:** нет (узкий охват; дизайн-документ ведёт основной ассистент по справочнику).
UI-реализация — `frontend-engineer`.

## Привязка (этап → инструмент)

| Этап | Инструмент | Что даёт |
|---|---|---|
| **3a план — визуальный дизайн** (только UI/носители; иначе пропуск) | `design-kb.md` + шаблон §4.5 | `docs/design.md`: **база** (бренд-ядро + UI-kit) + **расширения** (ось A масштаб · ось B носители) |
| 3a — ассеты по ходу | `document-skills:canvas-design` (постер/лого-концепт + design philosophy) · генераторы картинок по base-prompt из документа | визуальные ассеты в стиле |
| сопутствующие документы (презентации/бланки) | `document-skills:theme-factory` (быстрая тема) · `document-skills:brand-guidelines` (если нужен Anthropic-стиль) | оформление в корп-стиле |
| 5 код (стадии 2–6) | `frontend-design` + рубрика фронта (см. `rules/frontend.md`) | реализация документа в UI — исполняет фронт-колонка; дизайн-док здесь = спецификация, не исполнитель |

## Граница scope

- Создание/поддержка дизайн-документа — здесь (этап 3a, `design-kb`).
- Реализация в коде, доступность, перформанс, ревью UI — `frontend` (не дублируем).
- Глубина — по проекту: **база сразу** → ось A (Storybook/паттерны на масштабе продукта) → ось B
  (носители: презентация/PDF/визитка). Не вперёд продукта (`design-kb §3`).
