# Surface non-trivial assumptions — say them out loud before the answer

Перед ответом, который опирается на нетривиальную привязку — одной строкой проговори допущение.

## Триггер

Я готов выдать таблицу / список / вывод / план, в основе которого — привязка «X = Y», где:
- (а) Y **не сказан явно** пользователем в его сообщении;
- (б) Y **не очевиден** из имени X (т.е. `hero.mp4 = hero` очевидно; `section2 = страница 2` — нет).

## Действие

Перед основным ответом — одна строка вида:

> **Допущение: X = Y, проверяю.**

Дальше:

| Сложность проверки | Что делать |
|---|---|
| **1-2 tool calls** (один Grep / Read) | Сразу проверить и продолжить ответ |
| **Нетривиальная** (несколько файлов, web fetch, эксперимент) | Оставить строку допущения **перед** проверкой и дать пользователю шанс возразить, если он уже знает что Y ≠ то, что я думаю |
| **Принципиальная ветка** (от Y зависит всё дальше) | Спросить явно «правильно ли я понимаю, что X = Y», ждать ответа |

## Что НЕ требует строки допущения

- Привязка явно дана пользователем («посмотри файл `foo.py`») — это инструкция, не моё допущение.
- Имя файла буквально совпадает с ролью (`hero-video.mp4` = hero video).
- Привязка из прошлого сообщения в этом же разговоре, к которой нет сомнений.

## Почему

2026-05-28: я выдал таблицу параметров для `section2-*` как ответ на вопрос про страницу 2 лендинга. **Не пометил** связку «страница 2 = section2-*» как допущение — выдал тоном факта. Пользователь начал спорить с **цифрами** (потому что я их преподнёс как замеренные факты), хотя проблема была в **привязке**. Прошло два раунда уточнений, прежде чем выяснилось, что я смотрел не те файлы.

Одна строка «Допущение: страница 2 = section2-*, проверяю» сэкономила бы оба раунда — пользователь увидел бы привязку и сразу сказал «нет, page 2 = loop = section3-*».

Стоимость одной строки = низкая. Стоимость пропуска = два раунда уточнений и потеря доверия.
