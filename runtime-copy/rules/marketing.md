# Маркетинг сайта — инструменты по этапам

Привязка маркетинг-скиллов к **этапам процесса 0–7** (`dev-workflow-kb.md`; шкала «этапы», не стадии кода).

**Источник скиллов — внешний набор `coreyhaines31/marketingskills` (MIT).** Установка: склонируй репо
в удобный путь (далее `<MARKETING>`):
```bash
git clone https://github.com/coreyhaines31/marketingskills   # обновление — git pull
```
Скиллы НЕ устанавливаются в Claude — тело читается **Read'ом по пути** `<MARKETING>/skills/<имя>/SKILL.md`
(0 токенов до чтения). Три текстовых скилла можно сделать глобально доступными, скопировав в
`~/.claude/skills/`: `copywriting` · `copy-editing` · `marketing-psychology` (тогда зовутся как обычные
скиллы в любой сессии).

**Гейт контекста (первым делом):** до любого маркетинг-скилла в проекте должен существовать
`.agents/product-marketing.md` (12 секций: аудитория/боли/конкуренты/дифференциация/голос…).
Нет файла → сначала создать скиллом `product-marketing/SKILL.md`. Почти все скиллы набора читают его сами.

## Гейты

- События аналитики **названы до деплоя**; срабатывание проверено на стадии 3 этапа 5.
- `seo-audit` пройден перед финалом этапа 5 (мета/каноникалы/Core Web Vitals).
- Тексты страниц — глобальные `copywriting`/`copy-editing`; конверсионные рычаги — `marketing-psychology`.

## Границы scope

- Продвижение и кампании (реклама, рассылки/email/sms, social, PR, launch, каталоги, outbound/продажи,
  pricing-исследования, A/B-программа, lead-magnets/free-tools, разбор конкретного конкурента,
  годовой marketing-plan) — **вне системы**: владелец зовёт явно по пути из набора.
- `tools/` набора — справочник интеграций по запросу: `<MARKETING>/tools/REGISTRY.md`
  (110 сервисов: API/MCP/CLI) + `tools/clis/*.js` (64 готовых CLI, ключи только из env).
- Реализация на сайте — фронт/TS-колонки и их роль-агенты; БД — `dba`. Маркетинг-колонка без роль-агента (v1).
