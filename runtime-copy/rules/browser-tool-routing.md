# Browser tool routing — дешёвый дефолт для routine, спец-MCP для тяжёлого

Для браузерных задач выбирай инструмент по задаче: routine-драйвинг (открыть / прочитать / скриншот /
клик) — самым дешёвым по токенам инструментом; глубокий perf/память — спец-MCP. Ниже — рабочая
маршрутизация; **дефолтный routine-инструмент подставь свой** (см. примечание).

## Триггер

Задаче нужен браузер: открыть страницу, прочитать text/HTML/snapshot, скриншот, клик/заполнить форму, скрейп, визуальная проверка/дизайн-ревью лендинга, perf-аудит, отладка вёрстки.

## Действие — что выбрать

| Задача | Инструмент |
|---|---|
| Открыть страницу, прочитать text/snapshot/html, скриншот, клик/fill, скрейп, **визуальный QA** | дешёвый routine-драйвер (дефолт) |
| Lighthouse, performance-trace, heap/memory snapshot, глубокий network-разбор | **chrome-devtools-mcp** (спец-инструменты; routine-драйвер их не заменяет) |
| Флоу, уже завёрнутый в существующий Playwright-скилл (`webapp-testing`, кастомные скиллы) | тот скилл как есть — не переписывать |
| playwright-mcp | универсальный fallback |

**Разовый перехват:** если в задаче явно назван инструмент («через playwright», «через chrome-devtools») — это переопределяет дефолт для этого случая.

## Примечание (дефолтный routine-драйвер)

Дефолт «дешёвого routine-драйвера» зависит от твоей установки:

- **Playwright MCP** / **chrome-devtools MCP** — если поднят соответствующий MCP-сервер (берут ~1.5–2k токенов/вызов на protocol/schema).
- **CLI-браузер через Bash** — собственный bun/node-компилированный CLI, отдающий результат в stdout: **0 токенов протокола**, но требует личной установки бинарника (напр. в `~/.claude/bin/<твой-cli>`). У свежей установки его нет — тогда дефолт = Playwright / chrome-devtools MCP.

## Почему

Routine-драйвинг через MCP-браузер берёт ~1.5–2k токенов/вызов на schema/protocol (за сессию из 20 команд — 30–40k). CLI-браузер через stdout — 0 токенов протокола, но требует установки. `chrome-devtools-mcp` нужен для perf/памяти (Lighthouse, трейсы, heap) — это его спец-инструменты, дешёвый драйвер их не заменяет. Поэтому: routine — самым дешёвым доступным инструментом; perf/память — спец-MCP.
