# Система разработки — указатель (копируется в проект)

> **Назначение.** Подключает проект к внешней системе разработки (конвейер
> «идея → продукт → сопровождение»). Сама система НЕ копируется в проект — она одна, проекты
> ссылаются на неё. В новый проект копируй ТОЛЬКО этот файл.
>
> ⚙️ **Перед использованием замени** `<ПУТЬ-К-СИСТЕМЕ>` ниже на абсолютный путь к папке, куда ты
> склонировал этот репозиторий (напр. `C:/Users/<you>/dev-system` или `/home/<you>/dev-system`).

## Оглавление системы (подтягивается автоматически)

@<ПУТЬ-К-СИСТЕМЕ>/CLAUDE.md

> **Fallback (для Claude):** если выше НЕТ содержимого оглавления (раздела «Карта системы (что где
> лежит)») — внешний `@`-импорт в этом проекте не одобрен и молча пропущен. Тогда перед первой
> задачей разработки прочитай `<ПУТЬ-К-СИСТЕМЕ>/CLAUDE.md` инструментом Read и действуй по нему.

## Как пользоваться (для Claude)

При задачах разработки в этом проекте — следуй маршруту системы из оглавления выше, по намерению:
сырая идея → `0-ideya/` · PRD/дизайн/план → `1-plan/` · код → `2-realizaciya/` · ремонт/развитие →
`3-soprovozhdenie/`. Заходи в нужный раздел/справочник по намерению, не грузи всё сразу.
(Нумерация «этапов 0–7» — это детализация процесса внутри `dev-workflow-kb`, не папки.)

**Приоритет:** правила ЭТОГО проекта (проектный `CLAUDE.md`) важнее общих, где конфликтуют.

**Защита:** файлы системы (этот репозиторий + runtime в `~/.claude/`) Claude не меняет по своей
инициативе — только по прямой просьбе владельца (см. `~/.claude/rules/protect-dev-system.md`).

## Как подключить (для владельца, разово на проект)

- Если у проекта НЕТ своего `CLAUDE.md` → переименуй этот файл в `CLAUDE.md`.
- Если `CLAUDE.md` уже есть → добавь в него одну строку: `@dev-system.md`
- При первом запуске Claude Code один раз спросит одобрить импорт — подтверди.
